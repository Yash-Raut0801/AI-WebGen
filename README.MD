## рҹ§  About

AI-WebGen is a full MERN stack web application that lets users generate complete websites from a single line of text. Powered by **Google Gemini AI**, it intelligently detects the type of project required вҖ” static or full MERN stack вҖ” generates all necessary files, displays them in a **Monaco code editor** with a live file explorer, offers a **live preview** for static sites, and allows instant **ZIP download** of the entire project.

No templates. No drag-and-drop. Just describe what you want вҖ” and it builds it.

## рҹҡҖ Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) v18 or above
- [Git](https://git-scm.com/)
- [MongoDB Atlas](https://cloud.mongodb.com/) account (free tier works)
- [Google Gemini API Key](https://aistudio.google.com/app/apikey) (free)

### 1. Clone the Repository

```bash
git clone https://github.com/yourusername/AI-WebGen.git
cd AI-WebGen
```

### 2. Backend Setup

```bash
cd server
npm install
```

Create your `.env` file inside `server/` (see [Environment Variables](#-environment-variables)):

```bash
cp .env.example .env
```

Start the backend:

```bash
npm run dev
```

You should see:

```
Server running on port 8000
MongoDB Connected
```

### 3. Frontend Setup

Open a **new terminal**:

```bash
cd client
npm install
npm start
```

App opens at `http://localhost:3000` вң…

## рҹ”җ Environment Variables

Create a `.env` file inside the `server/` folder:

```env
PORT=8000
MONGO_URI=your_mongodb_connection_string
GEMINI_API_KEY=your_gemini_api_key
```

| Variable         | Description                         | Where to get it                                               |
| ---------------- | ----------------------------------- | ------------------------------------------------------------- |
| `PORT`           | Backend server port (default: 8000) | Set to any free port                                          |
| `MONGO_URI`      | MongoDB Atlas connection string     | Atlas вҶ’ Connect вҶ’ Drivers                                     |
| `GEMINI_API_KEY` | Google Gemini API key               | [aistudio.google.com](https://aistudio.google.com/app/apikey) |

> рҹ’Ў **Tip for Indian networks:** If `mongodb+srv://` throws `querySrv ECONNREFUSED`, use the **Non-SRV** connection string from Atlas вҶ’ Connect вҶ’ Drivers вҶ’ toggle _"I can't use a DNS SRV connection"_.

## рҹ“Ў API Reference

Base URL: `http://localhost:8000/api`

| Method   | Endpoint        | Description                           |
| -------- | --------------- | ------------------------------------- |
| `POST`   | `/generate`     | Generate a website from a text prompt |
| `GET`    | `/projects`     | Fetch all saved projects              |
| `GET`    | `/projects/:id` | Fetch a specific project by ID        |
| `DELETE` | `/projects/:id` | Delete a project                      |

## рҹ§  AI Prompt Engineering

The backend uses keyword-based intent detection to decide the output type **before** calling Gemini, ensuring the AI is given the right context and constraints.

**Triggers for MERN Stack generation:**

| Keyword                        | Detected Intent             |
| ------------------------------ | --------------------------- |
| `login` / `authentication`     | Needs backend auth routes   |
| `dashboard` / `admin panel`    | Needs protected routes + DB |
| `database` / `CRUD`            | Needs Mongoose models       |
| `API` / `REST`                 | Needs Express endpoints     |
| `register` / `user management` | Needs Users model           |

Everything else defaults to a **Static Website** (HTML + CSS + JS).
